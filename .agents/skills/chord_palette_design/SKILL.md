# Chord Palette Mobile UI Guidelines

翔吾さんとの過去のやり取りから得られた、Chord Palette の UI・UX 設計における最も重要なルールとパターンです。今後、UIを追加・変更する際は必ず以下の原則に従ってください。

## 1. パレットの基本デザイン（ピル形式と矢印）
- **ピル（Pill）型のコード表示**: パレット内のコードは、上段に「度数（Roman Numeral）」、下段に「コード名」を配置した縦積みのコンパクトな丸形デザイン（ピル）とする。
- **進行の視覚化（矢印）**: コード間には必ず `→`（矢印）を配置し、進行の繋がりを直感的にわかりやすくする。
- **機能ごとの色分け**: T（Tonic / 青）、SD（Subdominant / 橙・黄）、D（Dominant / 赤）の視覚的コーディングを適用し、コードの機能的役割が一目で判別できるようにする。

## 2. 編集・差し替えのUX（編集モードと個別削除）
- **直感的な「上書き」**: 完了・追加済みのコードをタップすると「編集モード（`.editing`）」に入る。この際、コードは青い太枠、パルスアニメーションで「浮き上がった」視覚表現にする。その状態で別のコードリストをタップすると、「末尾の追加」ではなく「選択中コードの差し替え」が行われる。
- **個別削除（✕ボタン）**: 編集モードに入った（選択された）コードには、右上に赤い「✕」ボタンが表示されるよう実装する。モバイルでのタップミスを防ぎつつ、不要になったコードだけをピンポイントで削除できる設計にする。
- **自動解除**: 差し替え完了後は自動で編集モードを解除し、スムーズな連続追加処理を可能にする。

## 3. モバイル特化の省スペースレイアウト
- **アクションボタンのヘッダー集約**: 「戻る（Undo）」「保存」「クリア」などの主要アクションボタンは、下部のボトムシート内ではなく、画面上部の**ヘッダー（ロゴ横）の右側エリア**にアイコン＋テキストの形でコンパクトに集約する。
- **History（履歴）の占有率圧縮**: 履歴が溜まってきても画面領域を圧迫しないよう、Historyエリアの縦幅を削り、無駄な余白を削減したレイアウトを徹底する。（必要に応じて個別削除ボタンを併設）
- **コントロール類のコンパクト化**: BPMやDrumの設定項目はスライダー等の画面幅を取るUIを避け、数値入力をベースとした横並び（コンパクトな縦型・横型）にまとめる。

## 4. プレミアムな操作感と視覚的フィードバック（WOW要素）
- **アニメーションの活用**: ユーザーアクションに対して、必ず微細なフィードバック（`hover`時の浮き上がり、`active`時の沈み込み、`pulse`アニメーション等）をCSSアニメーションで実装する。
- **視界の整理**: 不要になった見出しや冗長なテキスト（例："Palette", "BPM変更"など）はそぎ落とし、全体としてプレミアムな作曲アプリのように見せること。
